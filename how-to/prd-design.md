# PRD Rules: Design

## Descripción
Este documento contiene las reglas y guías para la creación de PRDs desde la perspectiva de Diseño.
Proporciona guías de alto nivel (nivel CDO) sobre el "cómo" del diseño, que sirven como base para que diseñadores o herramientas como Figma Make desarrollen propuestas detalladas.

## Reglas

- Reduce el número de pasos al usuario
- Informa al usuario en todo momento para fomentar la confianza
- Plantea siempre empty states cuando sea necesario
- Minimiza la carga cognitiva del usuario
- Prioriza la claridad sobre la complejidad
- Asegura feedback inmediato en todas las interacciones
- Diseña para el caso de uso más común primero
- Considera el contexto de uso del usuario
- Mantén la consistencia visual y de interacción
- Facilita la reversibilidad de acciones
- Limita las propuestas de diseño al alcance establecido en el PRD
- Presenta las consideraciones de forma narrativa y natural, describiendo el flujo del usuario y su experiencia, en lugar de listar explícitamente cada regla como checklist
