# How-to: Brief

## Descripción
Este documento contiene las reglas y guías para crear y gestionar briefs en el proyecto.

## Reglas

### Creación del Project Brief

1. **Pregunta al usuario**: "¿para qué hacemos el proyecto?"
2. **Espera la respuesta del usuario** antes de continuar
3. **Identifica el proyecto**: Si no está claro cuál es el proyecto en cuestión, pregunta al usuario
4. **Actualiza la descripción del proyecto en Linear**:
   - Ve al proyecto correspondiente en Linear
   - Actualiza la descripción del proyecto con el siguiente contenido (en formato Markdown):

```
# Project Brief

## ¿para qué hacemos el proyecto?

[aquí la respuesta del usuario]
```
